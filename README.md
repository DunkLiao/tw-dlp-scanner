# 🛡️ DLP 敏感資料掃描工具 v1.0.0 使用手冊

這是一套 Windows 桌面版敏感資料掃描工具，用來協助檢查指定資料夾內的文件是否含有個資、信用卡號、電子郵件、銀行存款帳號、加密檔案等敏感內容。

工具會將命中結果顯示在畫面中，也可以匯出成 Excel 報表，方便後續留存、人工確認或交付稽核使用。

## 👥 適合誰使用

- 需要在送出檔案前檢查是否含有敏感資料的使用者
- 需要掃描共享資料夾、下載資料夾、暫存資料夾的人員
- 需要產出 Excel 掃描紀錄供主管、稽核或資訊單位確認的人員
- 不熟悉指令操作、希望透過視窗介面完成掃描的使用者

## ✅ 掃描前準備

### 如果你拿到的是 EXE

直接執行 `DLPScanner.exe` 即可，不需要另外安裝 Python。

### 如果你拿到的是原始碼

請先確認電腦已安裝 Python，並在專案資料夾內執行：

```powershell
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
```

本工具使用的主要套件：

- `python-docx`
- `openpyxl`
- `pdfplumber`
- `xlrd`
- `olefile`
- `pyinstaller`

## 🚀 啟動方式

### 使用原始碼啟動

在專案資料夾中執行：

```powershell
python dlp_scanner.py
```

### 建置 Windows EXE

如需產生可攜式 Windows 執行檔，可執行：

```powershell
.\build_exe.bat
```

建置完成後，執行檔通常會出現在：

```text
dist\DLPScanner.exe
```

## 🧭 操作流程

1. 開啟工具。
2. 在「掃描資料夾」欄位確認要掃描的路徑。
3. 點選「瀏覽」可改選其他資料夾。
4. 視需要勾選「包含子資料夾」。
5. 視需要勾選「匯出後自動開啟 Excel」。
6. 點選「開始掃描」。
7. 等待進度列跑完。
8. 若有命中，結果會出現在下方清單。
9. 如需留存結果，點選「匯出 Excel」。

💡 建議先掃描範圍較小的資料夾確認結果，再掃描大型資料夾。

---

## 🖥️ 畫面欄位說明

掃描結果區會顯示以下欄位：

| 欄位 | 說明 |
|---|---|
| 風險 | 命中的風險等級，例如高、中、低、提示 |
| 處置 | 建議處置，例如僅記錄、記錄並阻擋、額外OR提示 |
| 檔案名稱 | 命中的檔案名稱 |
| 命中類型 | 命中的敏感資料類型 |
| 次數 | 該檔案中命中的筆數 |
| 命中範例 | 完整命中內容，最多顯示部分樣本 |
| 完整路徑 | 命中文件的完整位置 |

### 顏色判讀

- 🔴 高風險：建議優先處理
- 🟡 中風險：建議確認內容是否可外流
- 🟢 低風險：建議留存紀錄並視情況處理
- ℹ️ 提示：額外 OR 命中，需人工判斷是否為真正敏感資料

## 📁 支援掃描格式

| 類型 | 副檔名 |
|---|---|
| 文字類 | `.txt`, `.csv`, `.log`, `.md`, `.json`, `.xml`, `.html`, `.htm` |
| Word | `.doc`, `.docx` |
| Excel | `.xls`, `.xlsx` |
| PowerPoint | `.ppt` |
| PDF | `.pdf` |
| ZIP 壓縮檔 | `.zip` |

### ZIP 掃描

- 會掃描 ZIP 內的支援格式檔案。
- 支援巢狀 ZIP。
- ZIP 內加密項目會以加密檔案規則提示。
- 為避免超大型壓縮檔造成掃描時間過長，工具有深度與容量限制。

### 加密檔案偵測

工具會嘗試偵測以下可能加密的檔案：

- `.docx`
- `.xlsx`
- `.pptx`
- `.pdf`
- `.zip`
- `.rar`
- `.7z`

⚠️ `.rar` 與 `.7z` 目前只偵測為可能加密目標，不會解開內容掃描。

## 🔎 敏感資料規則

| 編號 | 類型 | 說明 |
|---|---|---|
| 1 | 組合式個資 | 同一檔案內符合指定個資組合條件 |
| 2 | 加密檔案 | 偵測 Office、PDF、ZIP 等可能加密檔案 |
| 3 | 台灣地址 | 偵測縣市、路、街、巷、弄、號等地址格式 |
| 4 | 中文姓名 | 搭配稱謂、職稱等線索判斷中文姓名 |
| 5 | 台灣身分證字號 | 偵測台灣身分證格式 |
| 6 | 護照號碼 | 偵測 9 碼數字格式 |
| 7 | 信用卡號 | 偵測 13 至 19 碼數字，並使用 Luhn 檢核 |
| 8 | 銀行存款帳號與關鍵字 | 正式命中需「客戶往來明細查詢」加 12 至 14 碼帳號 |
| 9 | 公文密等關鍵字組合 | 偵測密等、解密條件、保密期限等關鍵字組合 |
| 10 | 電子郵件 | 偵測 Email 格式 |
| 11 | 居留證號碼 | 偵測新版與舊版居留證格式 |

## 🚦 風險與處置說明

| 風險 | 畫面處置 | 建議使用方式 |
|---|---|---|
| 高 | 記錄並阻擋 | 優先確認，不建議直接外傳 |
| 中 | 僅記錄 | 建議人工確認內容與用途 |
| 低 | 僅記錄 | 可留存紀錄，視情境判斷 |
| 提示 | 額外OR提示 | 未必達正式門檻，但建議人工檢查 |

### 主要門檻

| 類型 | 低風險 | 中風險 | 高風險 |
|---|---:|---:|---:|
| 組合式個資 | 符合基本組合門檻 | 任一組合條件達 3 筆 | 任一組合條件達 5 筆 |
| 加密檔案 | N/A | N/A | 任一加密 Office、PDF 或 ZIP |
| 台灣地址 | 40 筆 | 60 筆 | 80 筆 |
| 中文姓名 | 40 筆 | 80 筆 | 100 筆 |
| 台灣身分證字號 | N/A | 1 筆 | 2 筆 |
| 護照號碼 | N/A | 1 筆 | 2 筆 |
| 信用卡號 | N/A | 1 筆 | 2 筆 |
| 銀行存款帳號與關鍵字 | N/A | N/A | 關鍵字加 12 至 14 碼帳號 |
| 公文密等關鍵字組合 | N/A | N/A | 命中指定關鍵字組合 |
| 電子郵件 | 40 筆 | 60 筆 | 80 筆 |
| 居留證號碼 | N/A | 1 筆 | 2 筆 |

## 🔔 額外 OR 命中提示

除了正式風險門檻外，工具也會執行額外 OR 檢核。

這代表：即使某些資料未達正式低、中、高風險門檻，只要有原始命中，仍會額外提醒使用者。

### 會出現在哪裡？

- 掃描完成後的 OR 提示視窗
- 掃描結果清單
- Excel 報表

### 顯示方式

額外 OR 命中會以以下方式呈現：

| 欄位 | 顯示 |
|---|---|
| 風險 | 提示 |
| 處置 | 額外OR提示 |
| 命中範例 | 實際命中內容 |

### 範例

- 單一 Email 未達 40 筆低風險門檻，仍會顯示 OR 提示。
- 只要偵測到 12 至 14 碼連續數字帳號，就會觸發銀行存款帳號 OR 提示。
- 銀行存款帳號若要成為正式高風險政策命中，仍需同時符合「客戶往來明細查詢」關鍵字。

---

## 💳 信用卡號檢核說明

信用卡號偵測不只看數字長度，還會使用 Luhn 演算法檢核。

這可以降低一般帳號、流水號、單號被誤判成信用卡號的機率。

## 🧩 常見問題

### Q1：掃描完成沒有資料，代表一定安全嗎？

不一定。工具只能偵測目前支援的文字與文件內容。如果檔案是圖片、掃描 PDF、截圖或 OCR 無法辨識的內容，可能無法被偵測。

### Q2：為什麼只有 OR 提示，沒有高風險？

OR 提示代表有原始敏感資料跡象，但未達正式政策門檻。例如單一 Email 或單一帳號格式可能只列為提示，需人工判斷。

### Q3：為什麼銀行帳號會出現提示？

目前 12 至 14 碼連續數字會觸發銀行存款帳號額外 OR 提示。這是提醒使用者人工確認，不代表一定是正式高風險。

### Q4：為什麼信用卡號沒有命中？

信用卡號會經過 Luhn 檢核。如果數字長度符合但檢核失敗，工具不會列為信用卡號。

### Q5：可以掃描 ZIP 裡面的檔案嗎？

可以。工具會掃描 ZIP 內的支援格式，也支援巢狀 ZIP。但過大或過深的 ZIP 可能因安全限制被略過。

### Q6：Excel 報表可以直接外寄嗎？

不建議直接外寄。

## ⚠️ 限制與注意事項

- 不支援圖片 OCR。
- 掃描式 PDF 若沒有可擷取文字，可能無法命中。
- `.doc`、`.ppt` 採最佳努力方式擷取文字，結果可能受檔案格式影響。
- `.rar`、`.7z` 不會解壓縮掃描內容。
- 正規表示式偵測可能產生誤判，重要結果仍需人工確認。
- 命中範例會顯示完整命中內容，請將報表視為敏感文件保存。
- 大型資料夾或大量壓縮檔可能需要較長掃描時間。

## 🛠️ 故障排除

### 無法匯出 Excel

請確認已安裝 `openpyxl`：

```powershell
pip install openpyxl
```

### PDF 無法讀取

請確認已安裝 `pdfplumber`：

```powershell
pip install pdfplumber
```

### 舊版 Excel `.xls` 無法讀取

請確認已安裝 `xlrd`：

```powershell
pip install xlrd
```

### 舊版 Office 檔案內容讀不到

`.doc`、`.ppt` 屬於舊版 Office 格式，工具會盡量擷取可讀文字。若結果不完整，建議先另存為 `.docx` 或 `.pptx` 後再掃描。

## 🔐 使用建議

- 掃描前先確認資料夾範圍，避免掃描不必要的大量檔案。
- 對高風險與 OR 提示項目進行人工複核。
- 匯出的 Excel 報表請放在受控資料夾。
- 不要把掃描報表直接寄給不相關人員。
- 若要保存稽核紀錄，建議以日期或案件名稱命名報表。
